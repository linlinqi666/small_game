### 核心规则
1.  最大循环次数：最多执行5轮，每轮计数，到第5轮后自动停止，输出最终结果
2.  提前终止：如果某轮检测后，确认代码无任何错误、功能完整、体验达标，可提前停止，不用跑完所有轮次
3.  每轮递进优化：每轮的检测深度逐步提升，不重复做无用检测，具体流程如下：
    --------------------------------------------------
    【第1轮：基础语法错误修复】
    优先解决最紧急的语法问题：
    1.  检查括号、引号、分号是否匹配，中英文符号混用
    2.  检查变量/函数未定义、空值调用、DOM获取失败
    3.  检查事件绑定错误、passive选项误用
    4.  检查基础语法报错，确保控制台无红叉错误
    发现问题立即修复，不影响原有功能。

    【第2轮：核心功能逻辑补全】
    确保拼图核心功能100%实现：
    1.  图片能正确切割成9块，无错位、无拉伸、无空白
    2.  移动端拼图块可正常拖动，不瞬移、不卡顿
    3.  拖动结束自动吸附到目标格子，吸附位置准确
    4.  已吸附的拼图块可随时取下重新拖动
    5.  初始拼图块散落在目标区域下方，无完全重叠
    缺失功能自动补全，修复逻辑BUG。

    【第3轮：兼容性与异常修复】
    解决运行异常与兼容问题：
    1.  检查touch/mouse事件兼容，确保微信浏览器正常运行
    2.  检查图片预加载，避免空白块、白屏
    3.  检查边界限制，避免拼图块被拖出页面
    4.  检查响应式适配，不同手机尺寸显示正常
    5.  检查错误处理，图片加载失败有友好提示
    发现问题立即修复，确保代码稳定。

    【第4轮：性能与体验优化】
    优化运行流畅度与用户体验：
    1.  优化touchmove事件，减少计算，保证拖动跟手
    2.  优化过渡动画，吸附过程平滑自然
    3.  优化初始散落位置，避免拼图块重叠过多
    4.  优化触摸区域，确保点击/拖动无盲区
    5.  优化资源大小，确保微信内秒开
    不破坏原有功能，只做体验提升。

    【第5轮：最终验收与收尾】
    做最终全量检测，输出验收报告：
    1.  全量检查所有语法、功能、兼容、体验问题
    2.  确认无新引入的BUG
    3.  输出最终的完整代码，以及验收报告
    --------------------------------------------------

### 输出要求
每轮输出必须包含：
1.  轮次标识：【第X轮检测优化】
2.  本轮发现的问题/优化点
3.  本轮修复/优化的具体内容
4.  修复后的完整代码
5.  若未到最大轮次、且还有问题，自动进入下一轮检测
